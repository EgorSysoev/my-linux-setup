GIT MANTRA: git init git add OR git add . git status

git commit -am "Commit message goes here."

git clone git://github.com/USER/REPOSITORY.git - download repo (later you keep refreshing with 'git pull origin master') -> You need SSH key. If you don't want, use https://github.com/USER/REPOSITORY.git for address.

GENERATE SSH KEY: 1. Check for existing keys: cd ~/.ssh; ls -al 2. Generate new key: ssh-keygen -t rsa -C "youremail@example.com" 3. Add your key to the ssh-agent: ssh-add ~/.ssh/idrsa 4. Add your key to GitHub: copy contents of ~/.ssh/id_rsa.pub and paste them into key field at Acount settings > SSH Keys > Add SSH key.

-- ADD TO GITHUB: Create remote repository on website git remote add origin git@github.com:/your-username/project-name.git git pull origin master git push origin master (git push --force) After your first push, you can simply type: git push

sometimes also: git push --set-upstream origin master

Also useful: git rm --cached mylogfile.log - untrack file without deleting it. git tag -a 0.9.1 -m "Version 0.9.1 release" - tag latest commit git tag - list local tags sudo chown -R yourname:yourgroup * - in .git/objects git reset --hard HEAD~1 - delete last commit git format-patch -1 - generate patch file git rev-parse HEAD - get sha of head git reset --hard - trash the changes git reset HEAD^ git revert

git describe - print version and hash of HEAD

Remote: git remote update - get info about state of remote git status -uno - check if everything up-to-date git remote show origin - print address of the origin git remote set-url origin git@github.com:myusername/my_repo.git - change the url of origin, ssh key needed

git remote set-url origin https://www.github.com/myusername/my_repo.git - same, but withouth key

Checkout: git log > ../gitLog - first save log to file git checkout - then checkout previous versions

git checkout head - return to head

gitstats

git log --name-only --author= - print changed files by commit

git ls-files - list files

Bisect: git bisect start git bisect bad - tell git that current version is bad git bisect good v25.0.2 - tell git the last good version you know about now git will checkout a version in between, so you can check it and tell: git bisect bad/good - this will countinue untill the first bad version is found git bisect reset - exit bisect mode
